PdfFile Class

 

Provides methods and properties that can be manipulated in order to create, view and save a PDF file.

TextFile

The name of an existing text file that is to be the source of the data for the PDF file. The file will be opened and read, and the number of columns in the PDF file will be based on the widest record encountered in the text file. You can still pass various other parameters to determine the overall characteristics of the resulting PDF file. When reading the text file, any form feed (ASCII 12) characters that are encountered in the text file will be translated to new page instructions when creating the PDF file.

 

Page Size

All pages in a PDF document are the same page size. The default page size is LETTER. You can change the page size used in a document by passing a custom page size value to the aPageSize parameter. Available page sizes are defined by the PdfPageSize enumeration.

 

Orientation

Pages in a PDF document default to portrait orientation. You can override the default orientation on a page by page basis as you add pages to the document. If you wish to change the default page orientation to landscape then you can do so by passing a value of PdfOrientation.Landscape to the aOrientation parameter.

 

Margins

The concept of page margins doesn't really exist in PDF files; it is generally left to the software that creates documents to determine if margins are required. For most applications a small margin works well so PdfFile defaults to applying default margins of 0.16" on all four sides of each page; this is about the same as the natural margins on most modern laser and ink-jet printers. If you wish to override the default margins you can do so by passing values to the aLeftMargin, aTopMargin, aRightMargin and aBottomMargin parameters. The values that you pass define the width or height of each margin in inches. Page margins only affect operations that use the row/column coordinates system, and have no affect on operations that use the absolute coordinates system.

 

Header and Footer Rows

Many reports use the concept of page header and footer rows so the PdfFile class allows you to reserve a number of rows for page header and/or footer information. You do this by passing the number of rows that you wish to reserve to the aHeaderRows and/or aFooterRows parameters. Headers and footers only affect operations that use the row/column coordinates system, and have no affect on operations that use the absolute coordinates system. For more information refer to Page Headers and Footers.

 

Columns and Font Size

When using the row/column coordinates system the page body will by default be divided into 80 columns across. You can override the number of columns by passing a custom value to the aMinimumColumns parameter. A font size will then be automatically selected in order to achieve the required number of columns within the page body, taking into account any left and right margins that have been defined.

 

If a relatively small number of columns is requested (e.g. 40 columns or less) then a fairly large font size will be selected. If you find that the automatically selected font size is too large for your requirements you can place a cap on the font size selection by passing a custom value to the aMaxFontSize parameter.

 

By default the font size that provides at least the number of columns requested will be selected, the page body will be configured to contain exactly the number of columns were requested, and if there is left over space then the left and right margins will be adjusted so that the page body appears to be centered horizontally in the page. Of course it is possible that the selected font size might actually be able to support slightly more columns that were requested. If you wish to leave the margins exactly as defined and have access to any additional columns that would fit in the page body then you can pass false to the aCenterBodyInPage parameter.

 

File Location and Partial Name

The location where the PDF file will be created is defined by the environment variable TEMP. You can override this default location by passing the optional aFileLocation parameter. You may pass either a full or relative file specification, or the name of an environment variable that defined the location to be used.

 

The name of the PDF file that will be created defaults to a time-stamped value of YYYY-MM-DD HH.MM.SS.pdf (e.g. 2015-08-31 11:47.03.pdf). You can customize this default name by passing the optional aPartialFileName parameter, any text that you pass will be added to the file name after the date and time stamp and before the file extension. You should pass a small number of descriptive words. For example, if you pass the value "Invoice 0072232" then the final file name would look something like this:

 

2015-08-29 12.16.51 Invoice 0072232.pdf

 

The partial file name value that you pass must not contain characters that are not valid in file specifications.

 

If you wish to totally customize the file name then you can set the FileName property to your custom file name after calling the constructor method.

 

Page Background Color

 

By default the background color of pages in a PDF document is white. You can override this by passing an alternate member of the PdfColor enumeration to the aBackgroundColor parameter.

 

Caution should be used when assigning page background colors because if the document is to be printed then using a non-white background color has the potential to use excessive amounts of printer ink or toner.

 

Extra Row Spacing

 

The aExtraRowSpacing parameter can be used to define that an amount of additional space should be added between each row when using the row/column coordinates system. Specify the amount of additional space in device independent pixels, and that amount of space will be added to the height of each row. The additional space is added below the actual space occupied by the select font size. This setting affects all rows, including header and footer rows. Bear in mind that specifying an amount of extra space between rows is likely to reduce the number of rows available in the page body.

DrawImage Method

 

Draws an image on the current page of the PDF document using absolute coordinates. The characteristics of the image are defined by individual passed parameters.

 

You may use JPEG (*.jpeg or *.jpg) or PNG (*.png) images, and for optimal results images should be created at a resolution of 72dpi.

 

public method DrawImage,          boolean

          required in  aFileSpec,     string

          required in  aX,            float

          required in  aY,            float

          optional in  aWidth,        float

          optional in  aHeight,       float

          optional out aErrorMessage, string

          endparams

 

You must at least specify the file spec of the image file to be drawn via the aFileSpec parameter as well as the location of the lower left pixel of the image via the aX and aY parameters, using the absolute coordinates system. You can optionally define the size of the image on the page by passing the optional aWidth and aHeight parameters. The values that you pass are a number of device independent pixels. Bear in mind that device independent pixels and image file pixels may not be the same thing. Device independent pixels in  PDF document are always use a resolution of 72dpi, whereas the pixels in an image file might be at a different resolution. For example, if you use an image file that was created at a resolution of 300dpi then the image will be displayed at a much larger size when added to a PDF file. The aWidth and aHeight parameters can be manipulated in order to correct these scaling problems in such cases.

 

The return value will be true on successful completion, or false if an error occurred. If the return value is false then an explanatory message can be obtained by passing a string variable to the aErrorMessage parameter.

NewPage Method

 

Completes the current page and adds a new page to the PDF document.

 

public method NewPage, int

          optional in aOrientation, PdfOrientation

          optional in aOutlineText, string

          endparams

 

When this method is called the first thing that happens is that the current page is completed. This includes the addition of any EndPage Auto Items to the page, as well as the addition of any header and/or footer content.

 

Once the current page has been completed a new page is added to the document. Having added a new page it is not possible to change the content of any previous page.

 

The orientation of the new page will default to the default page orientation of the document, unless you override it by passing a value to the optional aOrientation property. If the orientation changes then the values of the Height, Width, Rows and Columns properties will be updated to reflect the new page orientation. Note that the calculation of font size for the row/column coordinates system is NOT re-evaluated; the same font and font size will be used, resulting in a different number of rows and columns for pages in the alternate orientation.

 

If a document outline is being created then by default a new outline item will be added for each new page; the default outline text will be "Page <page number>". To override this default outline text pass a custom value to the aOutlineText parameter. To prevent the addition of an outline item for the new page pass a value of ^null to aOutlineText.

 

As soon as a page is added, any BeginPage Auto Items will be added to the new page.

 

The current Row and Column properties are both reset to a value of 1 and the PageNumber property is incremented.

Specifies an existing PDF file that is to be used to provide the page background for the PDF file currently being created. The PDF file specified might typically contain a "pre-printed" form of some kind, or perhaps a watermark image.

 

This method is only available on Windows, and can only be used if a licensed copy of PDFtk Pro is detected.

 

public method SetBackgroundFile, boolean

          required in  aPdfFile, string

          required out aErrorMessage, string

          endparams

 

Use the aPdfFile parameter to specify the name of a PDF file that contains a single page that is to be used as the background for all pages in the PDF file currently being created . A returns value of true indicates that the background file was successfully set. If there is a problem setting the background file, either because the specified file was not found, or because PDFtk Pro was not detected, then the method returns false, and also returns an explanatory error message via the aErrorMessage parameter.

 

SetPosition Method

 

Sets the current position used by the row/column coordinates system.

 

public method SetPosition, void

          optional in aRow,    int

          optional in aColumn, int

          endparams

 

Specify the new current position by passing values to the aRow and/or aColumn parameters. Calling this method causes the values of the Row and Column properties to be updated. You can also set the current position by manipulating those properties directly.
