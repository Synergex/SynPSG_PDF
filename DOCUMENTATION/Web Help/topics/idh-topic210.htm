General Concepts

 

A class named PdfFile represents a PDF file that you are creating. That class has various properties which are generally used to obtain information about the PDF file being created (although some properties allow you to change things), and also has various methods that generally are used to define the content for the PDF file.

 

Device Independent Pixels

 

All PDF files use the concept of device independent pixels to define the positioning and size of items rendered on pages within the file. Device independent pixels are square in shape and measure 1/72 of an inch in both dimensions. This means a standard US Letter sized page, which when held in a portrait orientation measures 8.5 inches wide and 11 inches high, has device a independent pixel resolution of 612 x 792.

 

Coordinates System
 

PDF files use an X / Y coordinates system where X values represent a position in the horizontal direction and Y values represent positions in the vertical direction. The pixel in the lower-left corner of each page has the coordinates 0,0, and the pixel in the upper-right corner has coordinates based on the height and width of the page in the chosen orientation. For example, the pixel at the upper-right corner of a Letter page, when displayed in a portrait orientation, has the coordinates 612,792.

 

Remember, device independent pixels are always 1/72 of an inch, so to determine the coordinates system for any given page take the size of the page in inches and multiply by 72.

Locating and Sizing Items

 

Ultimately the positioning and sizing of all items is based on the device independent pixels coordinates system, and indeed that system can be used, but to make some common operations easier the PdfFile class also provides for the ability to logically divide the page up into a grid of rows and columns (based on the size of a character when using a fixed-pitch font) and then use that grid as an alternate coordinates system.

 

Using Row / Column Coordinates

 

The row / column coordinates system is primarily intended for use when producing data-centric reports. It revolves around the fact that a single fixed-pitch font will be used to display the primary content in the document, although it is still possible to use other fonts and point sizes to display things like titles and headings. The Courier font is used because it is one of the very few native fonts that are supported by the PDF file specification; it must be inherently supported by all PDF applications and environments without the need to embed the font within the actual PDF document.

 

The point size that will be used with the row / column coordinates system is selected automatically based on a requested "minimum width" for the document expressed in terms of the number of columns needed to display the report. For example, a developer might specify that a width of 80 characters is needed to display a particular report, that the report should be displayed in portrait mode, and that each page must have have a 0.75 inch margin on both the left and right sides. Based on this information a particular point size will be selected that allows for at least the requested number of columns to be displayed within the page margins.

 

Once a point size has been selected in order to facilitate the required width in columns, the maximum number of rows per page can be calculated. This calculation takes into account any requested page top and bottom margins, and also any requested page header and page footer lines. These calculations all take place when the programmer creates a new PdfFile object, and the programmer can then access various information about the row/column coordinates system (e.g the width of the page in columns, and height of the page in rows) via various properties of the PdfFile object.

 

When working with the row/column coordinates system you can think of each page of a PDF document as being organized like this:

 

 

Pages can optionally have top, bottom, left and right margins, in which no content will be printed (unless you print content there via the absolute positioning system). The size of the margins are defined when you construct a new PdfFile object and once defined can't be changed.

 

Pages can also optionally have one or more header and / or footer rows defined. These are reserved areas that will contain content that will be added to all pages of the document. The number of header and / or footer rows is defined when you construct a new PdfFile object and once defined can't be changed. For more information refer to Working With Headers and Footers.

 

Whatever space remains after margins and page headers and footers have been allocated is allocated to the page body, which is then divided up into a grid of rows and columns. The number of rows available in the page body can be determined by referring to the Rows property, and the number of columns by referring to the Columns property.

 

The coordinates system used within the page body is based on a grid of rows and columns. The size of each cell (row/column intersection) is defined by the size of a character in the Courier font at the selected font size. The coordinates system is 1-based and is rooted in the top-left cell within the page body, coordinates 1,1. The highest row that you can address is defined by the Rows property and the highest column that you can address is defined by the Columns property.

 

The row / column coordinates system is always available, as is the absolute coordinates system. Both may be used at the same time.

 

Using Absolute Coordinates

 

The absolute positioning system provides total control over the positioning and sizing of items on the page; you will work with the native coordinates system of the PDF file and can size and position items without restriction. When working with the absolute coordinates system you can think of each page of a PDF document as being organized like this:

 

 

As you can see there are no predefined margins, headers or footers when working with the absolute coordinates system; the entire page width (X coordinates) and height (Y coordinates) is available to you to draw items.

 

The coordinates system is based on the width and height of the page expressed in terms of device independent pixels. The coordinates system is 0-based and is rooted in the bottom-left pixel of the page, coordinates 0,0. The highest X coordinate that you can address is defined by the Width property and the highest Y coordinate that you can address is defined by the Height property.

 

When positioning items such as text and images using the absolute coordinates system you express the position by specifying the X / Y coordinates of the bottom left corner of the item.

 

The absolute coordinates system is always available, as is the row / column coordinates system. Both may be used at the same time.
