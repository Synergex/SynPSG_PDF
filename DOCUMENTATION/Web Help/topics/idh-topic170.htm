Working with Fonts

 

PDF Implicit Fonts

 

The Adobe PDF specification defines a set of fonts that must be inherently supported by all applications that process PDF files. These fonts are:

 

Fixed-Pitch Fonts

Variable-Pitch Fonts

Symbol Fonts

Courier

Courier-Bold

Courier-Oblique

Courier-BoldOblique

Helvetica

Helvetica-Bold

Helvetica-Oblique

Helvetica-BoldOblique

Times-Roman

Times-Bold

Times-Italic

Times-BoldItalic

Symbol

ZapfDingbats

 

The advantage of using these implicitly supported fonts is that it is not necessary to embed the font within each PDF document that uses the font, because the font must be provided and supported by all applications that will process the document. This in turn means that the PDF documents are smaller, faster to transmit, etc.

 

By default the PdfFile class works with the implicit Courier font. When the PdfFile constructor method is called a required number of columns is defined (for working with the row/column coordinates system) and various point sizes of the Courier font are tried in order to determine the most appropriate font size that will deliver at least the required number of columns. The Courier font is used because it is the only implicit font that is a fixed-pitch font, allowing the page to be divided into rows and columns of an equal size.

 

You can change the current font at any time by calling PdfFile.SelectFont, and having done so all subsequent text drawing will use the requested font. To revert to the default font you can call PdfFile.ResetFont.

 

If you select a variable-pitch font then you can still display text via the methods that use the row/column coordinates system, but the text that you display may occupy more or less space than the same text would occupy if you were using the fixed-pitch font.

 

Here is some example code that displays text using an alternate implicit font:

 

;;Create an instance of the PdfFile class

pdf = new PdfFile()

 

;;Draw text in the default font

pdf.DrawTextRC("This text is displayed using the default Courier font.",1,1)

 

;;Switch to the implicit Helvetica font, same font size as we're currently using

pdf.SelectFont("Helvetica",pdf.FontSize)

 

;;Draw text in the alternate font

pdf.DrawTextRC("This text is displayed using the Helvetica font.",2,1)

 

;;Switch back to the default font

pdf.ResetFont()

 

;;Show the results

pdf.Preview()

 

TrueType Fonts

 

In addition to working with the implicit fonts you also have the option of loading TrueType fonts from a .ttf file. To load a TrueType font you call PdfFile.LoadTrueTypeFont which loads the font from the font file, embeds the font into the PDF document, and returns the internal name of the font. You can then pass that internal name to PdfFile.SelectFont in order to select the font for use, or you can pass the name to various methods that accept a font name, for example PdfFile.DrawText. To verify is a particular font is available for use call PdfFile.FontAvailable.

 

Here is some example code that displays text using an explicitly loaded and embedded TrueType font:

 

;;Create an instance of the PdfFile class

pdf = new PdfFile()

 

;;Load the TrueType font

arialFont = pdf.LoadTrueTypeFont("C:\Windows\Fonts\arial.ttf")

 

;;Switch to Arial 16 point

pdf.SelectFont(arialFont,16)

 

;;Draw text in Arial 16 point

pdf.DrawTextRC("This text is displayed using the Arial 16 point.",1,1)

pdf.DrawTextRC("So is this!",2,1)

 

;;Switch back to the default font

pdf.ResetFont()

 

;;Draw text, absolute positioning now, still using the Arial font, this time 12 point, Green

pdf.DrawText("This text is displayed using the Arial 12 point.",50,400,PdfColor.Green,arialFont,12)

 

;;Show the results

pdf.Preview()

 

 

IMPORTANT: If you transfer font files (*.ttf) from Windows to a Linux or OpenVMS system you must do so via a binary FTP transfer. Transferring fonts by copying them via a Samba or another file share (even as part of a ZIP file) probably won't result in the file having the correct file attributes on the target system. On OpenVMS systems font files must have the following attributes, which you can check via a DIRECTORY/FULL command:

 

File organization:  Sequential

Record format:      Fixed length 512 byte records
