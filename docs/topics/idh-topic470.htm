Working with Page Backgrounds

 

On Windows only, the PDF API allows you to provide background content for a new PDF file that you are creating by providing an existing PDF file that contains the background information to be used for each page of the new file. This is done by calling the SetBackgroundFile method.

 

Note that using this feature requires that a licensed copy of PDFtk Pro is present in your system PATH.

 

The background PDF file might typically contain the layout of a "pre-printed" form, or perhaps a watermark image. Using a background file has several potential benefits, including but not limited to:

 

1.Improved performance because the background file only needs to be built once, not for every page of every document produced.

2.Increased flexibility because the background PDF file can be produced by a wide variety of applications. For example you might use Microsoft Word or Excel to produce a background PDF file, and then use that file with the PDF API.

 

Here is a simple example of how to specify the use of a background file:

 

;;Create an instance of the PdfFile class

pdf = new PdfFile()

 

;;Name the other PDF file that defines page background content

if (!pdf.SetBackgroundFile("FORMS:DeliveryTicketForm.pdf",errorMessage)

          throw new Exception(errorMessage)

 

;;Other code to define the content of the PDF file

 

 

 

;;Show the results

pdf.Preview()
