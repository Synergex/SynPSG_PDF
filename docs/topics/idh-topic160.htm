Working with Images

 

When working with images you always work with the absolute coordinates system.

 

Displaying Images via Absolute Coordinates

 

The absolute coordinates system is based on the physical dimensions of the page expressed in device independent pixels. Any margins or page headers and/or footers that you may have configured do not apply when working with absolute coordinates.

 

The PdfFile class has specific methods that allow you to work with images and the absolute coordinates system. These methods are:

 

•DrawImage

•DrawImageObject

 

Here is some example code that displays an image using absolute coordinates:

 

;;Create an instance of the PdfFile class

pdf = new PdfFile()

 

;;Display a Synergy/DE logo towards the top of the page

pdf.DrawImage("sdelogo.png",200,700)

 

;;Show the results

pdf.Preview()

 

IMPORTANT: If you transfer images from Windows to a Linux or OpenVMS system you must do so via a binary FTP transfer. Transferring image files by copying them via a Samba or another file share (even as part of a ZIP file) probably won't result in the image file having the correct file attributes on the target system. On OpenVMS systems image files must have the following attributes, which you can check via a DIRECTORY/FULL command:

 

File organization:  Sequential

Record format:      Fixed length 512 byte records
