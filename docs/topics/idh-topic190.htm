Page Headers and Footers

 

The PdfFile class supports optional page header and footer regions which are areas at the the top and/or bottom of each page that are reserved for text that will appear on every page. The header and/or footer text can be defined once and is then automatically added to each page.

 

Page headers and footers are defined in terms of the number of rows that each will occupy at the top and/or bottom of each page, and these rows are effectively reserved from the number of rows available on the page; essentially reducing the number of rows allocated to the page body.

 

Page headers are printed below the top margin and above the main body of the page. Page footers are printed below the main body of the page and above the bottom margin.

 

No additional blank space is allocated between the bottom of the page headers and the top of the page body, or the bottom of the page body and the top of the page footers. If you require a physical separation between the page body and the page headers and footers then allocate one or more additional header or footer lines and leave them blank.

 

Enabling Page Headers and Footers

 

Page headers and footers are allocated by passing optional parameters to the constructor method of the PdfFile class. For example:

 

import SynPSG.PDF

 

main PdfTest

    record

        pdf,        @PdfFile

        headerRows, int

        footerRows, int

    endrecord

proc

    headerRows = 2

    footerRows = 2

    pdf = new PdfFile(,,,,,headerRows,footerRows)

   

   

    pdf = ^null

endmain

 

Defining Page Header Text

 

To define the text to be printed in the reserved page header lines you load the text into the HeaderText property. This property is an array of string and is initialized to the appropriate number of elements when you call the constructor method (above). For example:

 

pdf.HeaderText[1] = "Customer Phone List"

pdf.HeaderText[2] = ""

 

When setting the page header text you should ensure that the text that you specify does not exceed the width of the page, which you can determine by referring to the value of the Columns property. If you specified that a minimum number of columns must be available then you can rely on the fact that at least that number of columns will be available.

 

If you want one of the header lines to be blank it is not necessary to set it to an empty string like in the example above. This code was written to show how to address the various page header lines by their element number.

 

Defining Page Footer Text

 

To define the text to be printed in the reserved page footer lines you load the text into the FooterText property. This property is an array of string and is initialized to the appropriate number of elements when you call the constructor method (above). For example:

 

pdf.FooterText[1] = ""

pdf.FooterText[2] = "That's the end of this page, get ready for the next one..."

 

When setting the page footer text you should ensure that the text that you specify does not exceed the width of the page, which you can determine by referring to the value of the Columns property. If you specified that a minimum number of columns must be available then you can rely on the fact that at least that number of columns will be available.

 

If you want one of the footer lines to be blank it is not necessary to set it to an empty string like in the example above. This code was written to show how to address the various page header lines by their element number.

 

Using Replacement Tokens

 

There are several replacement tokens that can be used within the text of page header and/or footer lines. These tokens cause some value to be printed in place of the token. The tokens are:

 

DD/MM/YYYY

Prints the current date in DD/MM/YYYY format

[FILE]

Prints the name of the PDF file being created.

HH:MM

Prints the current time in HH:MM format.

HH:MM:SS

Prints the current time in HH:MM:SS format.

[LINE]

Prints a horizontal line across the current header or footer line. The line will be 1 device independent pixel thick and will be drawn in the current StrokeColor.

[LOCATION]

Prints the location where the PDF file will be saved.

MM/DD/YYYY

Prints the current date in MM/DD/YYYY format.

[PAGE#]

Prints the current page number.

 

This is an example of using replacement tokens:

 

pdf.HeaderText[1] = "Customer Phone List                   Printed MM/DD/YYYY at HH:MM"

pdf.HeaderText[2] = "[LINE]"

 

pdf.FooterText[1] = "[LINE]"

pdf.FooterText[2] = "Page [PAGE#]"

 

Changing Header or Footer Text

 

Header and footer text is added to a page at the very end of the processing of the page, at the time that the NewPage method is called to add the next page. This means that you may change the page HeaderText or FooterText at any time until the NewPage method is called, and your changes will take effect for the current page and all subsequent pages. It is not possible to change the header or footer text for previously completed pages and it is not possible to change the number of header or footer rows being used.

 

Alternate Headers and Footers

 

A PDF document can contain pages that are both portrait and landscape orientations. Clearly this could present a problem when using header or footer lines, because the width of the text in header or footer lines may be appropriate for the default orientation, but may be too narrow (or too wide) for pages in the alternate orientation. To address this the PdfFile class allows you to define alternate header and footer text for pages that may be added in the alternate orientation. To do this you must set the UseAlternateHeaders to true and then populate HeaderTextAlternate, or set UseAlternateFooters to true and then populate FooterTextAlternate. For example:

 

headerRows = 2

footerRows = 2

pdf = new PdfFile(,,,,,headerRows,footerRows)

 

pdf.HeaderText[1] = "Customer Phone List        Printed MM/DD/YYYY at HH:MM"

pdf.HeaderText[2] = "[LINE]"

 

pdf.FooterText[1] = "[LINE]"

pdf.FooterText[2] = "Page [PAGE#]"

 

pdf.UseAlternateHeaders = true

pdf.HeaderTextAlternate[1] = "Customer Phone List                             Printed MM/DD/YYYY at HH:MM"

pdf.HeaderTextAlternate[2] = "[LINE]"

 

In this example the regular page headers would be used for pages in the default orientation (portrait in this case) and the alternate page headers, containing wider text, would be used for pages using the alternate (landscape) orientation.
