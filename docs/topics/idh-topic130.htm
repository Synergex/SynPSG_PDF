Working with Text

 

There are two ways to work with text, either via the row/column coordinates system or via the absolute coordinates system. These two mechanisms are not mutually exclusive; you can work with both if you need to, even within a single page.

 

Displaying Text via Row/Column Coordinates

 

The row/column coordinates system is based on the minimum number of columns that you defined when you called the constructor method. If you didn't specify a minimum number of columns then the default value of 80 would have been used. Based on this minimum number of columns, and other factors such as margins and page orientation, a fixed-pitch font size is then chosen to best deliver the required page parameters.

 

When working with the row/column coordinates system you are always working with this automatically selected font and point size. You do have the option of modifying the font slightly by specifying things like bold, italic, or both, but you are always working with the same base font.

 

The PdfFile class has specific methods that allow you to work with text and the row/column coordinates system. These methods are:

 

•DrawTextRC

•DrawTextObjectRC

•NewLine

•SetPosition

 

Here is some example code that displays text using row/column coordinates:

 

;;Create an instance of the PdfFile class

pdf = new PdfFile()

 

;;Draw text at the current position

pdf.DrawTextRC("Hello world")

 

;;Move to the beginning of the next row

pdf.NewLine()

 

;;Draw text at the current position AND move to the beginning of the next row

pdf.DrawTextRC("Hello again",,,,,,true)

 

;;Draw text at a specific location

pdf.DrawTextRC("This text is at row 5 column 10",5,10)

 

;;This time make it bold and red

pdf.DrawTextRC("Bold red text",7,10,true,,PdfColor.Red)

 

;;Show the results

pdf.Preview()

 

Displaying Text via Absolute Coordinates

 

The absolute coordinates system is based on the physical dimensions of the page expressed in device independent pixels. Any margins or page headers and/or footers that you may have configured do not apply when working with absolute coordinates. By default you still work with the native Courier font and the automatically selected font size, but you also have the option of loading and using alternate fonts.

 

The PdfFile class has specific methods that allow you to work with text and the absolute coordinates system. These methods are:

 

•DrawText

•DrawTextObject

 

Here is some example code that displays text using absolute coordinates:

 

;;Create an instance of the PdfFile class

pdf = new PdfFile()

 

;;Draw text at a specific location

pdf.DrawTextRC("Hello world",50,500)

 

;;This time make it red

pdf.DrawTextRC("Red text",50,480,PdfColor.Red)

 

;;Show the results

pdf.Preview()

 

Auto Text Items

 

Auto text items are text objects that you can define once and that will then be added automatically to each page. For additional information refer to working with auto items.
