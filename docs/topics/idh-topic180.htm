Completing Documents

 

Once you have finished building the document you have two options. Both involve saving the PDF file to disk, and one also launches the file for preview by the user.

 

Previewing the PDF

 

To preview the document you call PdfFile.Preview. For example:

 

pdf = new PdfFile()

.

.

.

pdf.Preview()

 

The preview method first saves the PDF file to disk (based on the values of the PdfFile.FileLocation and PdfFile.FileName, and then launches the saved file using its default Windows file association. So if the .pdf file extension is associated with the Adobe Reader application, the application will start and display your final document.

 

After calling PdfFile.Preview the PdfFile object is no longer valid.

 

Saving the PDF

 

If you don't wish to have the user preview the document you can simply save it to disk by calling PdfFile.Save method. For example:

 

pdf = new PdfFile()

.

.

.

pdf.SaveToFile()

 

After calling PdfFile.Save the PdfFile object is no longer valid.

 

Invalidating the PdfFile Object

 

It is important to understand that calling either PdfFile.Preview or PdfFile.Save executes code that cleans up and deallocates resources. After calling either of these methods the PdfFile object is no longer valid.

 

Aborting PDF Creation

 

If for any reason you decide to stop working on the creation of a PDF file after creating an instance of the PdfFile class, you should call PdfFile.Dispose to clean up and release allocated resources. For example:

 

pdf = new PdfFile()

.

.

.

pdf.Dispose()
